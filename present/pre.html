LAUNCHER LIBRARIES

UI & Assets

  • Tkinter: Standard GUI toolkit used to create the start menu window.
  • PIL (Pillow): Handles image loading and resizing for the background.
  • Pygame: Utilized primarily for the audio mixer (Music/SFX).

System & Logic

  • Subprocess: Launches the independent game script ("bird2.py").
  • Math: Calculates sine waves for the "hovering" animation.
  • Sys: Provides access to the system path for the executable.

1. MAIN MODULE

The Game Engine

This is the entry point. It initializes the Pygame window, sets the frame rate, and runs the central Game Loop.

while True:
  check_events()
  update_screen()
  clock.tick(60)

2. BIRD MODULE

The Player Class

Handles the physics. Gravity is constantly applied to the Y-velocity, pulling the bird down every frame.

def move(self):
  self.vel += gravity
  self.rect.y += self.vel

3. SCORE MODULE

Score Handling

Tracks the player's score by incrementing a counter each time the bird successfully passes through a pipe pair.

if bird_rect.x > pipe.x:
  if not pipe.scored:
    score += 1
    pipe.scored = True

4. PIPES MODULE

Obstacle Generator

Spawns new pipes at random heights at fixed intervals and handles their movement to the left.

def create_pipe():
  height = random.choice(h)
  pipes.append(Pipe(x, h))

5. COLLISION MODULE

The Referee

Uses Pygame's built-in colliderect to check for overlaps between the Bird's hitbox and Pipes/Ground.

if bird_rect.colliderect(pipe):
  game_active = False
  return "GAME OVER"

6. SOUND MODULE

Audio Feedback

Manages the loading and playback of WAV files for specific game events (Flap, Score, Crash).

flap_fx = mixer.Sound('s.wav')
if jump_triggered:
  flap_fx.play()

GAME FLOW

State Management

  • Idle: Game loop running, waiting for first input.
  • Active: Gravity applies, pipes generate and move.
  • Game Over: Loop continues but updates stop, waiting for reset.
